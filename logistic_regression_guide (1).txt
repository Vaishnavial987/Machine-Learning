        LOGISTIC REGRESSION - SIMPLE GUIDE FOR BEGINNERS
                    (Every Value Explained)

PART 1: WHAT IS LOGISTIC REGRESSION?
================================================================================

Simple Definition:
A way to predict YES/NO answers based on information you have.

Real-World Example:
Will a student PASS (YES=1) or FAIL (NO=0) the exam based on:
- Hours studied
- Previous exam scores
- Attendance

That's it! We're predicting 0 or 1.

================================================================================
PART 3: WEIGHTS - WHAT ARE THEY? WHERE DO THEY START?
================================================================================

Simple Definition:
Weights are IMPORTANCE scores. They tell us how much each feature matters.

Real Example:
If "Hours studied" has weight = 0.5
If "Previous score" has weight = 0.3
If "Attendance" has weight = 0.2

It means "Hours studied" is MORE IMPORTANT than previous score.

INITIAL WEIGHTS (Where they start):
We start with ZERO weights:
w0 (bias/intercept) = 0
w1 (weight for Hours) = 0
w2 (weight for Prev Score) = 0
w3 (weight for Attendance) = 0

WHY ZERO?
Because we don't know yet which features are important. Training will fix them.

VISUAL:
Initial State Before Training:
┌──────────────┬──────────┐
│ Weight Name  │ Value    │
├──────────────┼──────────┤
│ w0 (bias)    │ 0        │
│ w1 (Hours)   │ 0        │
│ w2 (Prev Sc) │ 0        │
│ w3 (Attend)  │ 0        │
└──────────────┴──────────┘

These zeros will change during training!


================================================================================
PART 4: THE MAGIC FORMULA - HOW DO WE PREDICT?
================================================================================

Step 1: COMBINE EVERYTHING
Simple Definition:
Multiply each feature by its weight and add them up.

Formula:
w0 - > baseline/bias
z = w0 + (w1 × Feature1) + (w2 × Feature2) + (w3 × Feature3)

WHERE DOES THIS COME FROM?
This is called a "linear combination". It's like:
- w0 = base score (bias)
- w1 × Feature1 = how much Feature 1 contributes
- w2 × Feature2 = how much Feature 2 contributes
- w3 × Feature3 = how much Feature 3 contributes

EXAMPLE - ROW 2 (Student who PASSED):
z = w0 + (w1 × 3) + (w2 × 55) + (w3 × 70)

BUT WAIT - our weights are all 0, so:
z = 0 + (0 × 3) + (0 × 55) + (0 × 70) = 0

Now we have a number 'z'. But we need a PROBABILITY (0 to 1, like 50%).


Step 2: CONVERT TO PROBABILITY USING SIGMOID
Simple Definition:
A special curve that converts ANY number to a value between 0 and 1.

Formula:
prediction = 1 / (1 + e^(-z))

WHERE DOES THIS COME FROM?
'e' is a special number = 2.71828... (Euler's number)
^(-z) means "power of -z"
1 / ( ) means "1 divided by"

So the formula says:
1. Take z
2. Multiply by -1 (make it negative: -z)
3. Calculate e^(-z)
4. Add 1 to it
5. Divide 1 by the result

WHY THIS FORMULA?
Mathematicians proved this curve has nice properties for probability problems.

SIMPLE EXAMPLE:
If z = 0:
prediction = 1 / (1 + e^(-0))
           = 1 / (1 + e^0)
           = 1 / (1 + 1)        [because e^0 = 1]
           = 1 / 2
           = 0.5 (50% chance)

If z = 2:
prediction = 1 / (1 + e^(-2))
           = 1 / (1 + 0.135)     [because e^(-2) ≈ 0.135]
           = 1 / 1.135
           = 0.88 (88% chance)

If z = -2:
prediction = 1 / (1 + e^(-(-2)))
           = 1 / (1 + e^2)
           = 1 / (1 + 7.389)     [because e^2 ≈ 7.389]
           = 1 / 8.389
           = 0.12 (12% chance)

UNDERSTANDING:
- Big positive z → prediction close to 1 (confident YES)
- Big negative z → prediction close to 0 (confident NO)
- z near 0 → prediction near 0.5 (unsure)

PART 5: MAKING OUR FIRST PREDICTION
================================================================================

Let's make a REAL prediction with Row 2 data:

Student Info:
Hours = 3, Previous Score = 55, Attendance = 70, Actual Answer = 1 (Passed)

Current Weights (still zeros):
w0 = 0, w1 = 0, w2 = 0, w3 = 0

Step 1: Calculate z
z = w0 + (w1 × 3) + (w2 × 55) + (w3 × 70)
z = 0 + (0 × 3) + (0 × 55) + (0 × 70)
z = 0

Step 2: Calculate prediction using sigmoid
prediction = 1 / (1 + e^(-0))
           = 1 / (1 + 1)
           = 0.5

Step 3: What does this mean?
We're saying this student has 50% chance of passing.

Step 4: Is this correct?
Actual answer = 1 (student DID pass)
Our prediction = 0.5 (we said 50%)

ERROR = |Actual - Prediction| = |1 - 0.5| = 0.5

This is WRONG! We predicted 50% but student passed (100% should happen).
Our weights are bad (they're all zeros). We need to UPDATE them!


================================================================================
PART 6: MEASURING HOW WRONG WE ARE (COST FUNCTION)
================================================================================

Simple Definition:
A score showing how bad our predictions are. Lower score = better predictions.

Formula:
Cost = -[y × log(prediction) + (1-y) × log(1-prediction)]

WHERE DOES THIS COME FROM?
This is called "Binary Cross-Entropy Loss".
'log' is natural logarithm (ln). It's a mathematical function.

WHY THIS FORMULA?
When y = 1 (should be YES):
- If prediction = 0.99 → cost is very small (we were right!)
- If prediction = 0.01 → cost is very large (we were very wrong!)

When y = 0 (should be NO):
- If prediction = 0.01 → cost is very small (we were right!)
- If prediction = 0.99 → cost is very large (we were very wrong!)

EXAMPLE - Row 2:
y = 1 (student passed)
prediction = 0.5 (we said 50%)

Cost = -[1 × log(0.5) + (1-1) × log(1-0.5)]
     = -[1 × log(0.5) + 0 × log(0.5)]
     = -[1 × (-0.693) + 0]
     = -[-0.693]
     = 0.693

WHERE DOES log(0.5) = -0.693 COME FROM?
log(0.5) is the natural logarithm of 0.5.
Using calculator: ln(0.5) ≈ -0.693

WHAT DOES COST = 0.693 MEAN?
Cost of about 0.7 is BAD. Our prediction was quite wrong.

Let's calculate for all 5 students with current bad weights:

ALL PREDICTIONS WITH WEIGHT = 0:
All students get z = 0, so prediction = 0.5 for everyone.

┌──────┬────────┬──────────┬─────────────┬────────────────────┐
│ Row  │Actual y│Prediction│Cost Formula │ Cost Value         │
├──────┼────────┼──────────┼─────────────┼────────────────────┤
│ 1    │   0    │   0.5    │-[0×log(0.5) │                    │
│      │        │          │+1×log(0.5)] │ 0.693              │
│      │        │          │             |                    │
│ 2    │   1    │   0.5    │-[1×log(0.5) │                    │
│      │        │          │+0×log(0.5)] │ 0.693              │
│      │        │          │             |                    │
│ 3    │   0    │   0.5    │-[0×log(0.5) │                    │
│      │        │          │ +1×log(0.5)]│ 0.693              │
│      │        │          │             |                    │
│ 4    │   1    │   0.5    │-[1×log(0.5) │                    │
│      │        │          │+0×log(0.5)] │ 0.693              │
│      │        │          │             |                    │
│ 5    │   1    │   0.5    │-[1×log(0.5) │                    │
│      │        │          │+0×log(0.5)] │ 0.693              │
├──────┼────────┼──────────┼─────────────┼────────────────────┤
│ AVERAGE (TOTAL COST)                   │ 0.693              │
└──────┴────────┴──────────┴─────────────┴────────────────────┘

Average Cost = (0.693 + 0.693 + 0.693 + 0.693 + 0.693) / 5 = 0.693

So our model has COST = 0.693 (very bad! We want cost near 0)


================================================================================
PART 7: HOW TO IMPROVE? - GRADIENT DESCENT
================================================================================

Simple Definition:
A method to find which direction to change weights so cost goes DOWN.

Analogy:
Imagine you're on a hill blindfolded. You can feel the slope under your feet.
- Steep slope → walk more aggressively
- Gentle slope → walk gently
This is how gradient descent works.

Basic Idea:
1. Calculate how much each weight CAUSED the error
2. Change that weight in the opposite direction
3. Repeat until cost stops decreasing


================================================================================
PART 8: CALCULATING GRADIENT (ERROR DIRECTION)
================================================================================

Simple Definition:
Gradient tells us: "Which direction should we change this weight?"

Formula:
gradient = (1/number_of_samples) × Σ(prediction - actual) × feature

WHERE DOES THIS COME FROM?

Step-by-step breakdown:
1. (prediction - actual) = ERROR for one sample
2. × feature = how much this feature contributed to error
3. Σ = sum all of these across all samples
4. ÷ number_of_samples = average (so we don't bias toward large datasets)

EXAMPLE - Calculate gradient for w1 (Hours weight):

┌────┬──────────┬──────────────────┬────────┬──────────────────────┐
│Row │Prediction│(pred - actual)   │Hours  │(pred-actual)×Hours   │
├────┼──────────┼──────────────────┼────────┼──────────────────────┤
│ 1  │  0.5     │ 0.5 - 0 = 0.5    │  2    │ 0.5 × 2 = 1.0        │
│ 2  │  0.5     │ 0.5 - 1 = -0.5   │  3    │ -0.5 × 3 = -1.5      │
│ 3  │  0.5     │ 0.5 - 0 = 0.5    │  2    │ 0.5 × 2 = 1.0        │
│ 4  │  0.5     │ 0.5 - 1 = -0.5   │  5    │ -0.5 × 5 = -2.5      │
│ 5  │  0.5     │ 0.5 - 1 = -0.5   │  4    │ -0.5 × 4 = -2.0      │
├────┼──────────┼──────────────────┼────────┼──────────────────────┤
│    │          │                  │  SUM   │ 1.0 - 1.5 + 1.0 -2.5 │
│    │          │                  │        │  - 2.0 = -4.0        │
└────┴──────────┴──────────────────┴────────┴──────────────────────┘

Gradient for w1 = -4.0 / 5 = -0.8

WHERE DOES EACH VALUE COME FROM?

Row 1:
- Prediction = 0.5 (we calculated this earlier)
- Actual = 0 (student failed)
- Error = 0.5 - 0 = 0.5 (we predicted too high)
- Hours = 2 (from dataset)
- Contribution to gradient = 0.5 × 2 = 1.0

Row 2:
- Prediction = 0.5
- Actual = 1 (student passed)
- Error = 0.5 - 1 = -0.5 (we predicted too low)
- Hours = 3 (from dataset)
- Contribution = -0.5 × 3 = -1.5

(Same logic for rows 3, 4, 5)

Sum all contributions = 1.0 - 1.5 + 1.0 - 2.5 - 2.0 = -4.0
Divide by number of samples = -4.0 / 5 = -0.8

WHAT DOES GRADIENT = -0.8 MEAN?
Negative gradient means: "Increase this weight! It will reduce error."

If gradient was positive, it would mean: "Decrease this weight!"


================================================================================
PART 9: UPDATING THE WEIGHT
================================================================================

Simple Definition:
Change the weight in the opposite direction of the gradient.

Formula:
new_weight = old_weight - (learning_rate × gradient)

WHERE DOES THIS COME FROM?

Step-by-step:
1. We have old_weight = w1 = 0
2. Gradient = -0.8 (tells us direction)
3. Learning_rate = 0.5 (how big a step to take. We choose this!)
4. We move opposite to gradient direction

EXAMPLE:
old_w1 = 0
gradient_w1 = -0.8
learning_rate = 0.5

new_w1 = 0 - (0.5 × (-0.8))
       = 0 - (-0.4)
       = 0 + 0.4
       = 0.4

WHERE DID 0.4 COME FROM?
- old weight was 0
- We multiply gradient (-0.8) by learning rate (0.5) = -0.4
- Subtract from old weight: 0 - (-0.4) = +0.4
- New weight is 0.4

WHY learning_rate = 0.5?
It's a choice we make! Common values: 0.01, 0.1, 0.5
- Small learning rate → slow training (more iterations needed)
- Large learning rate → fast training (might overshoot)
- We usually try different values

DOING THIS FOR ALL WEIGHTS (after iteration 1):

For w0 (bias):
gradient_w0 = (average of all errors) = (0.5 - 0.5 + 0.5 - 0.5 - 0.5) / 5 = -0.1
new_w0 = 0 - (0.5 × (-0.1)) = 0 + 0.05 = 0.05

For w2 (Previous Score weight):
┌────┬─────────────────┬──────────┬────────────────────────────┐
│Row │(pred - actual)  │Prev Score│(pred-actual)×PrevScore     │
├────┼─────────────────┼──────────┼────────────────────────────┤
│ 1  │ 0.5             │ 40       │ 0.5 × 40 = 20              │
│ 2  │ -0.5            │ 55       │ -0.5 × 55 = -27.5          │
│ 3  │ 0.5             │ 45       │ 0.5 × 45 = 22.5            │
│ 4  │ -0.5            │ 75       │ -0.5 × 75 = -37.5          │
│ 5  │ -0.5            │ 65       │ -0.5 × 65 = -32.5          │
├────┼─────────────────┼──────────┼────────────────────────────┤
│    │                 │ SUM      │ 20 - 27.5 + 22.5 - 37.5    │
│    │                 │          │ - 32.5 = -54.5             │
└────┴─────────────────┴──────────┴────────────────────────────┘

gradient_w2 = -54.5 / 5 = -10.9
new_w2 = 0 - (0.5 × (-10.9)) = 0 + 5.45 = 5.45

For w3 (Attendance weight):
┌────┬─────────────────┬────────────┬───────────────────────────┐
│Row │(pred - actual)  │Attendance  │(pred-actual)×Attendance   │
├────┼─────────────────┼────────────┼───────────────────────────┤
│ 1  │ 0.5             │ 50         │ 0.5 × 50 = 25             │
│ 2  │ -0.5            │ 70         │ -0.5 × 70 = -35           │
│ 3  │ 0.5             │ 60         │ 0.5 × 60 = 30             │
│ 4  │ -0.5            │ 85         │ -0.5 × 85 = -42.5         │
│ 5  │ -0.5            │ 80         │ -0.5 × 80 = -40           │
├────┼─────────────────┼────────────┼───────────────────────────┤
│    │                 │ SUM        │ 25 - 35 + 30 - 42.5 - 40  │
│    │                 │            │ = -62.5                   │
└────┴─────────────────┴────────────┴───────────────────────────┘

gradient_w3 = -62.5 / 5 = -12.5
new_w3 = 0 - (0.5 × (-12.5)) = 0 + 6.25 = 6.25

AFTER ITERATION 1:
┌──────────────┬─────────────┬────────────┐
│ Weight       │ Old Value   │ New Value  │
├──────────────┼─────────────┼────────────┤
│ w0 (bias)    │ 0           │ 0.05       │
│ w1 (Hours)   │ 0           │ 0.4        │
│ w2 (Prev Sc) │ 0           │ 5.45       │
│ w3 (Attend)  │ 0           │ 6.25       │
└──────────────┴─────────────┴────────────┘

IMPORTANT: These weights have changed! Now our predictions will be different!


================================================================================
PART 10: MAKING BETTER PREDICTIONS WITH NEW WEIGHTS
================================================================================

Now let's predict again using NEW weights:
w0 = 0.05, w1 = 0.4, w2 = 5.45, w3 = 6.25

PREDICTING ROW 2 AGAIN:
Hours = 3, Prev Score = 55, Attendance = 70

z = w0 + (w1 × 3) + (w2 × 55) + (w3 × 70)
  = 0.05 + (0.4 × 3) + (5.45 × 55) + (6.25 × 70)
  = 0.05 + 1.2 + 299.75 + 437.5
  = 738.5

prediction = 1 / (1 + e^(-738.5))
           = 1 / (1 + almost 0)
           ≈ 1.0 (99.9% chance)

BEFORE: We predicted 0.5 (50%)
NOW: We predict 1.0 (99.9%)
ACTUAL: 1 (student passed)

Much better! We're getting closer to the right answer!

We would repeat this many times (maybe 100 or 1000 iterations) until:
- Weights stop changing much
- Cost gets very low
- Predictions become accurate


================================================================================
PART 11: L1 REGULARIZATION (LASSO) - SIMPLE VERSION
================================================================================

SIMPLE PROBLEM:
With normal training, weights can become very large. This causes:
- Model gets too confident
- Overfitting (memorizes data instead of learning patterns)
- Weights for less important features don't shrink to zero

SOLUTION: Add a PENALTY for large weights!

L1 PENALTY IDEA:
"If your weight is big, we punish you by adding to the cost!"

FORMULA:
new_cost = old_cost + λ × (sum of absolute values of weights)

WHERE:
- λ (lambda) = penalty strength (we choose this, like 0.1 or 1.0)
- |w| = absolute value (always positive, even if w is negative)

EXAMPLE:
Say we have weights: w0=0.05, w1=0.4, w2=5.45, w3=6.25
Old cost = 0.5
λ = 0.1

L1_penalty = λ × (|0.05| + |0.4| + |5.45| + |6.25|)
           = 0.1 × (0.05 + 0.4 + 5.45 + 6.25)
           = 0.1 × 12.15
           = 1.215

new_cost_with_L1 = 0.5 + 1.215 = 1.715

COST INCREASED! Why do we want this?
Because now the algorithm will try to SHRINK the weights to reduce cost!

WHAT HAPPENS WITH L1:
If a weight is not helping much, it gets pushed all the way to ZERO.
This is like saying: "This feature is useless, remove it!"

EXAMPLE AFTER MANY ITERATIONS WITH L1:
Old weights: w0=0.05, w1=0.4, w2=5.45, w3=6.25

With L1 penalty, maybe we get:
w0 = 0.05   (still useful bias)
w1 = 0      (feature 1 got eliminated!)
w2 = 4.5    (feature 2 reduced)
w3 = 5.8    (feature 3 reduced)

So L1 AUTOMATIC SELECTS FEATURES!

GRADIENT WITH L1:
When calculating gradient for updating, we add L1 penalty gradient:

new_gradient = old_gradient + λ × sign(w)

WHERE sign(w) = +1 if w is positive, -1 if w is negative

EXAMPLE - Update w2 with L1:
old_gradient = -10.9
λ = 0.1
w2 = 5.45 (positive)
sign(w2) = +1

L1_gradient_part = 0.1 × 1 = 0.1
new_gradient = -10.9 + 0.1 = -10.8

new_w2 = old_w2 - (learning_rate × new_gradient)
       = 5.45 - (0.5 × (-10.8))
       = 5.45 + 5.4
       = 10.85

The weight INCREASES, but the L1 penalty makes sure it doesn't grow too much.

SUMMARY OF L1:
- Good at: Removing useless features (making model simple)
- Best for: When you have many features and want automatic selection
- How: Adds absolute value penalty to cost


================================================================================
PART 12: L2 REGULARIZATION (RIDGE) - SIMPLE VERSION
================================================================================

DIFFERENT PROBLEM WITH L1:
Sometimes L1 removes TOO MANY features. It's too aggressive.
What if we want all features to stay but just be smaller?

SOLUTION: Use L2 - a gentler penalty

L2 PENALTY IDEA:
"If your weight is big, we square it and add that penalty!"

FORMULA:
new_cost = old_cost + λ × (sum of squared weights)

WHERE:
- λ (lambda) = penalty strength
- w² = weight squared (always positive)

EXAMPLE:
Weights: w0=0.05, w1=0.4, w2=5.45, w3=6.25
Old cost = 0.5
λ = 0.1

L2_penalty = λ × (0.05² + 0.4² + 5.45² + 6.25²)
           = 0.1 × (0.0025 + 0.16 + 29.7 + 39.06)
           = 0.1 × 68.92
           = 6.892

new_cost_with_L2 = 0.5 + 6.892 = 7.392

WHAT HAPPENS WITH L2:
All weights get shrunk, but NONE become exactly zero.

EXAMPLE AFTER MANY ITERATIONS WITH L2:
Old weights: w0=0.05, w1=0.4, w2=5.45, w3=6.25

With L2 penalty, maybe we get:
w0 = 0.04    (slightly smaller)
w1 = 0.25    (smaller but NOT zero)
w2 = 4.2     (smaller but NOT zero)
w3 = 5.1     (smaller but NOT zero)

All features stay! L2 doesn't do feature selection.

GRADIENT WITH L2:
new_gradient = old_gradient + λ × 2 × w

(The "2" comes from calculus - derivative of w² is 2w)

EXAMPLE - Update w2 with L2:
old_gradient = -10.9
λ = 0.1
w2 = 5.45

L2_gradient_part = 0.1 × 2 × 5.45 = 1.09
new_gradient = -10.9 + 1.09 = -9.81

new_w2 = old_w2 - (learning_rate × new_gradient)
       = 5.45 - (0.5 × (-9.81))
       = 5.45 + 4.905
       = 10.355

The weight changes less aggressively than without L2.

SUMMARY OF L2:
- Good at: Keeping all features, preventing overconfidence
- Best for: When all features are important
- How: Adds squared weight penalty to cost


================================================================================
PART 13: ELASTIC NET (L1 + L2) - SIMPLE VERSION
================================================================================

NEW PROBLEM:
L1 removes too much. L2 keeps everything.
What if we want something in between?

SOLUTION: Combine both L1 and L2!

ELASTIC NET FORMULA:
new_cost = old_cost + λ × [α × (sum of |w|) + (1-α) × (sum of w²)]

WHERE:
- λ = total penalty strength
- α = mixing parameter (0 to 1)
  - α = 1.0 → acts like L1 (removes features)
  - α = 0.0 → acts like L2 (keeps all features)
  - α = 0.5 → 50% L1, 50% L2 (middle ground)

EXAMPLE:
Weights: w0=0.05, w1=0.4, w2=5.45, w3=6.25
Old cost = 0.5
λ = 0.1
α = 0.5

L1_part = 0.1 × 0.5 × (0.05 + 0.4 + 5.45 + 6.25)
        = 0.05 × 12.15
        = 0.6075

L2_part = 0.1 × 0.5 × (0.0025 + 0.16 + 29.7 + 39.06)
        = 0.05 × 68.92
        = 3.446

ElasticNet_penalty = 0.6075 + 3.446 = 4.0535

new_cost = 0.5 + 4.0535 = 4.5535

WHAT HAPPENS WITH ELASTIC NET:
Some weak features get removed (L1 effect).
Important features stay but are smaller (L2 effect).
Middle ground between L1 and L2.

EXAMPLE AFTER MANY ITERATIONS WITH ELASTIC NET:
Old weights: w0=0.05, w1=0.4, w2=5.45, w3=6.25

With ElasticNet penalty (α=0.5), maybe we get:
w0 = 0.04    (slightly smaller - L2 effect)
w1 = 0       (eliminated - L1 effect)
w2 = 4.8     (smaller but kept - balanced)
w3 = 5.5     (smaller but kept - balanced)

Some features gone, some kept!

GRADIENT WITH ELASTIC NET:
new_gradient = old_gradient + λ × [α × sign(w) + (1-α) × 2 × w]

EXAMPLE - Update w2 with ElasticNet:
old_gradient = -10.9
λ = 0.1
α = 0.5
w2 = 5.45 (positive)
sign(w2) = +1

L1_part = 0.1 × 0.5 × 1 = 0.05
L2_part = 0.1 × 0.5 × 2 × 5.45 = 0.545
new_gradient = -10.9 + 0.05 + 0.545 = -10.305

new_w2 = 5.45 - (0.5 × (-10.305))
       = 5.45 + 5.1525
       = 10.6025

SUMMARY OF ELASTIC NET:
- Good at: Balancing feature selection and stability
- Best for: When you're not sure which features matter
- How: Combines L1 penalty (remove weak) and L2 penalty (stabilize)


================================================================================
PART 14: COMPARING ALL THREE METHODS (VISUAL)
================================================================================

================================================================================
PART 17: KEY FORMULAS - QUICK REFERENCE
================================================================================

WITHOUT REGULARIZATION:
─────────────────────

1. Prediction (sigmoid):
   prediction = 1 / (1 + e^(-z))
   where z = w0 + w1×x1 + w2×x2 + w3×x3

2. Cost (Binary Cross-Entropy):
   cost = -[y × log(prediction) + (1-y) × log(1-prediction)]

3. Gradient:
   gradient = (1/m) × Σ(prediction - actual) × feature

4. Weight Update:
   new_weight = old_weight - (learning_rate × gradient)


L1 REGULARIZATION:
──────────────────

1. Cost with L1:
   cost = base_cost + λ × Σ|w|

2. Gradient with L1:
   gradient = data_gradient + λ × sign(w)

3. Effect: Some weights become exactly ZERO


L2 REGULARIZATION:
──────────────────

1. Cost with L2:
   cost = base_cost + λ × Σ(w²)

2. Gradient with L2:
   gradient = data_gradient + λ × 2 × w

3. Effect: All weights shrink but don't become zero


ELASTIC NET:
────────────

1. Cost with ElasticNet:
   cost = base_cost + λ × [α × Σ|w| + (1-α) × Σ(w²)]

2. Gradient with ElasticNet:
   gradient = data_gradient + λ × [α × sign(w) + 2×(1-α) × w]

3. Effect: Some weights shrink to zero, others stay


================================================================================
PART 18: IMPORTANT PARAMETERS YOU CHOOSE
================================================================================

LEARNING RATE:
What is it: Size of step to take when updating weights
Range: Usually 0.001 to 1.0
Effect:
- Too small (0.0001): Training is very slow
- Too large (1.0): Might jump over the solution
- Good start: 0.1 or 0.01
Example: We used 0.5 (pretty aggressive)

NUMBER OF ITERATIONS:
What is it: How many times to repeat the training loop
Range: Usually 10 to 10,000
Effect:
- Too few: Model hasn't learned enough
- Too many: Wastes time, little improvement
- Good practice: Look at cost, stop when it stops decreasing

LAMBDA (λ) - REGULARIZATION STRENGTH:
What is it: How much to penalize large weights
Range: Usually 0.001 to 10
Effect:
- Too small (0.001): Almost no regularization
- Too large (10): Over-penalizes, weights become too small
- Good start: 0.1 or 1.0

ALPHA (α) - FOR ELASTIC NET ONLY:
What is it: Balance between L1 and L2
Range: 0 to 1
Effect:
- α = 0: Pure L2
- α = 0.5: 50% L1, 50% L2
- α = 1: Pure L1
- Good start: 0.5


================================================================================
PART 19: COMMON MISTAKES AND HOW TO AVOID
================================================================================

MISTAKE 1: Using weights that are too large
Wrong: Initialize weights to random(100 to 1000)
Right: Initialize to 0 or small random values like random(-0.01 to 0.01)
Why: Large initial weights cause unstable training

MISTAKE 2: Learning rate too high
Wrong: learning_rate = 1.0
Right: learning_rate = 0.01 or 0.1
Why: You'll jump over the solution and training fails

MISTAKE 3: Not using regularization with many features
Wrong: Training with 50 features, no regularization
Right: Use L1, L2, or ElasticNet with 50 features
Why: Large number of features causes overfitting

MISTAKE 4: Regularization too strong
Wrong: λ = 100 with only 10 features
Right: λ = 0.1 or 1.0 with 10 features
Why: Over-regularization makes model too simple and bad

MISTAKE 5: Not standardizing features
Wrong: Feature 1 ranges 0-100, Feature 2 ranges 0-1000000
Right: Scale all features to same range
Why: Large numbers dominate, small numbers ignored

HOW TO SCALE FEATURES:
scaled_feature = (feature - mean) / standard_deviation

This makes all features have mean=0 and spread=1


================================================================================
PART 20: COMPLETE TRAINING ALGORITHM (STEP-BY-STEP)
================================================================================

INPUT:
- Training data with features X and labels y
- learning_rate = 0.1
- λ (lambda) = 0.1
- num_iterations = 100
- regularization_type = 'L1' or 'L2' or 'ElasticNet' or 'None'

ALGORITHM:

Step 1: INITIALIZE WEIGHTS
──────────────────────────
w0 = 0
w1 = 0
w2 = 0
... (all weights start at 0)

Step 2: SCALING (Optional but recommended)
──────────────────────────────────────────
For each feature:
   feature_mean = average of all values in that feature
   feature_std = standard deviation
   feature_scaled = (feature - feature_mean) / feature_std

Step 3: LOOP FOR num_iterations TIMES (e.g., 100 times)
───────────────────────────────────────────────────────

   FOR iteration = 1 TO 100:

      a) FORWARD PASS - Make predictions
      ─────────────────────────────────
         FOR each sample i from 1 to m:
            z_i = w0 + w1×x1_i + w2×x2_i + w3×x3_i
            prediction_i = 1 / (1 + e^(-z_i))
         END FOR

      b) CALCULATE COST
      ─────────────────
         base_cost = average of [-y × log(prediction) + (1-y) × log(1-prediction)]
         
         IF regularization == 'None':
            total_cost = base_cost
         
         ELSE IF regularization == 'L1':
            penalty = λ × (|w0| + |w1| + |w2| + |w3|)
            total_cost = base_cost + penalty
         
         ELSE IF regularization == 'L2':
            penalty = λ × (w0² + w1² + w2² + w3²)
            total_cost = base_cost + penalty
         
         ELSE IF regularization == 'ElasticNet':
            penalty = λ × [0.5 × (|w0|+|w1|+|w2|+|w3|) 
                         + 0.5 × (w0²+w1²+w2²+w3²)]
            total_cost = base_cost + penalty
         
         PRINT "Iteration", iteration, "Cost =", total_cost

      c) CALCULATE GRADIENTS FOR EACH WEIGHT
      ───────────────────────────────────────
         FOR weight wj:
            data_grad = (1/m) × Σ(prediction_i - y_i) × x_j_i
            
            IF regularization == 'None':
               total_grad = data_grad
            
            ELSE IF regularization == 'L1':
               l1_grad = λ × sign(wj)  [+1 if positive, -1 if negative]
               total_grad = data_grad + l1_grad
            
            ELSE IF regularization == 'L2':
               l2_grad = λ × 2 × wj
               total_grad = data_grad + l2_grad
            
            ELSE IF regularization == 'ElasticNet':
               l1_grad = λ × 0.5 × sign(wj)
               l2_grad = λ × 0.5 × 2 × wj
               total_grad = data_grad + l1_grad + l2_grad
            
            gradient_wj = total_grad
         END FOR

      d) UPDATE ALL WEIGHTS
      ─────────────────────
         FOR each weight wj:
            wj = wj - (learning_rate × gradient_wj)
         END FOR

   END FOR (next iteration)

Step 4: RETURN FINAL WEIGHTS
────────────────────────────
   RETURN w0, w1, w2, w3, ...

Step 5: PREDICT ON NEW DATA
───────────────────────────
   FOR new_sample:
      z = w0 + w1×new_x1 + w2×new_x2 + w3×new_x3
      prediction = 1 / (1 + e^(-z))
      
      IF prediction > 0.5:
         output = 1 (YES)
      ELSE:
         output = 0 (NO)
   END FOR

PART 22: HOW TO KNOW WHEN TO STOP TRAINING
================================================================================

TRACKING COST OVER ITERATIONS:

Iteration 0:   Cost = 0.693
Iteration 1:   Cost = 0.683
Iteration 2:   Cost = 0.672
Iteration 5:   Cost = 0.618
Iteration 10:  Cost = 0.501
Iteration 20:  Cost = 0.312
Iteration 50:  Cost = 0.045
Iteration 100: Cost = 0.0001

GOOD TRAINING:
- Cost decreases each iteration
- Decrease gets smaller and smaller
- Eventually flattens out

BAD TRAINING:
- Cost doesn't decrease
- Cost increases
- Jumps up and down wildly

STOPPING RULES:

1. EARLY STOPPING:
   Stop when cost doesn't improve for 10 iterations in a row
   (Prevents overfitting)

2. THRESHOLD:
   Stop when cost < 0.0001
   (Model is good enough)

3. MAX ITERATIONS:
   Stop at 1000 iterations
   (Time limit - don't train forever)

4. CONVERGENCE:
   Stop when weight changes are < 0.0001
   (Weights stopped changing)
